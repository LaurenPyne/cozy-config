# cozy-config

"Cozy" is an opinionated, modular NixOS configuration for my primary workstation.

This repository is designed around how I work, not around being a universal framework. Every decision reflects my own workflow, preferences, and tools—from the desktop environment and terminal to the development languages and editor.

Modularity in "Cozy" means separating responsibilities, not removing opinions. Each module has a single purpose and can be understood, modified, or replaced independently. Reusability is a natural consequence of good modular design, not the primary objective.

As the system grows, new functionality is added as focused modules that fit into a clear structure. This keeps the configuration easy to navigate, maintain, and extend while preserving a coherent vision for the machine it powers.

"Cozy" is intended to evolve over time. The Git history documents that evolution, and the repository serves not only as my system configuration but also as a record of the design decisions that shaped it.


## Goals

- Modular flake layout
- Home Manager
- GNOME desktop
- Ghostty terminal
- Everforest theme
- VS Code development
- Rust, Python, Lua and PICO-8 environments
- Bubblewrap sandboxing
- OpenClaw isolation

Built one commit at a time.
